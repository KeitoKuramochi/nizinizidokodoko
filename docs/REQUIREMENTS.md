# REQUIREMENTS

> このファイルは Planner が生成します。手動で編集しないでください。

## 画面一覧

### ホーム画面（唯一の画面）

アプリ起動時に表示される単一の画面。画面遷移はない。

**表示内容**:
- アプリタイトル（例: 「NiziPro」または「虹方角チェッカー」）
- 位置情報取得中のローディング表示
- 虹の方角を示すコンパス UI（方位角を矢印または回転ディスクで表現）
- 虹の方位角の数値（例: 「北東 45°」）
- 気象条件バッジ（「虹が出やすい」「出にくい」）
- 気象条件の補足説明（例: 「雨が降っており太陽が出ています」「雲が多く太陽が確認できません」）
- エラー時のメッセージ（位置情報拒否・API取得失敗など）

## 機能一覧

### F-01: 位置情報取得

ブラウザ Geolocation API を使って現在地（緯度・経度）を取得する。

- ページ読み込み時に自動で許可を求める
- 許可されるまでローディング表示を出す
- 拒否された場合は「位置情報の許可が必要です」メッセージを表示する
- 取得エラーの場合もメッセージを表示する

### F-02: 太陽位置計算

SunCalc.js を使って現在地・現在時刻から太陽の方位角（azimuth）と仰角（altitude）を計算する。

- 計算はクライアントサイドのみで完結する
- 仰角が 0° 以下（太陽が地平線以下）の場合は「夜間のため虹は出ません」と表示する

### F-03: 虹の方角計算

太陽方位角から虹が見える方角を計算する。

- 虹は反太陽点方向（太陽と正反対の方向）を基準に表示する
- 方位角を「北 0°」基準で 0〜360° の数値で保持する
- 表示時は 8 方位（北・北東・東・南東・南・南西・西・北西）のラベルも併記する

### F-04: 気象データ取得

Open-Meteo API（`https://api.open-meteo.com/v1/forecast`）を使って現在地の気象データを取得する。

取得するパラメータ:
- `precipitation`（降水量 mm/h）
- `cloud_cover`（雲量 %）

判定ロジック:
- 降水量 > 0 かつ 雲量 < 80 → 「虹が出やすい」
- それ以外 → 「出にくい」
- 雲量 >= 80 の場合 → 「曇りのため太陽が見えにくい」
- 降水量 = 0 の場合 → 「雨が降っていません」

### F-05: コンパス UI 表示

虹の方角を視覚的に示す UI を表示する。

- 方位角に応じて矢印またはインジケーターが回転する
- 上が「北」になっている方位盤（またはそれに相当する表示）
- 数値（方位角）と 8 方位ラベルを文字でも表示する

### F-06: 気象条件表示

気象条件の判定結果をバッジ＋補足テキストで表示する。

- 「虹が出やすい」は緑系のバッジ
- 「出にくい」は灰色または赤系のバッジ
- 理由テキスト（降水量・雲量の状態を自然な日本語で説明）

### F-07: エラーハンドリング

各取得・計算フェーズのエラーをユーザーに分かりやすく表示する。

- 位置情報拒否: 「位置情報の許可が必要です。ブラウザの設定から許可してください。」
- API 取得失敗: 「気象データの取得に失敗しました。太陽位置のみ表示しています。」
- 夜間: 「現在夜間のため、虹は出ません。」
- エラーでも太陽位置計算は引き続き表示できる場合は表示を維持する

## ユーザーフロー

```
1. アプリをブラウザで開く
2. 「位置情報を取得中...」のローディングが表示される
3. ブラウザが位置情報の許可を求めるダイアログを表示する
   [拒否] → エラーメッセージを表示して終了
   [許可] → 4へ進む
4. SunCalc.js で太陽位置を即時計算する
5. 太陽が地平線以下の場合 → 「夜間のため虹は出ません」を表示して終了
   太陽が出ている場合 → 6へ進む
6. 虹の方角を計算してコンパス UI に表示する
7. 同時に Open-Meteo API に気象データをリクエストする
8. 気象データ取得中は気象条件欄を「取得中...」とする
9. 取得完了 → 気象条件バッジと補足テキストを表示する
   取得失敗 → エラーメッセージを表示する（コンパスは維持）
```

## 非機能要件

- スマートフォン幅（375px〜）で表示が崩れないこと
- タブレット・PC 幅（768px〜）でも見やすく表示されること
- 位置情報取得後、コンパス表示まで 3 秒以内に表示が完了すること（API 取得は非同期で構わない）
- `npm run build` がエラーなく完了すること
- TypeScript の型エラーが 0 件であること
- `any` 型・`@ts-ignore` を使わないこと

## 制約・除外事項

- バックエンドサーバーを立てない
- データベースを使わない
- ユーザー認証・ログイン機能は作らない
- ユーザー投稿・SNS 連携は作らない
- デバイスの物理コンパスセンサー（DeviceOrientationEvent）は使わない
- Open-Meteo 以外の外部 API は使わない
- APIキー・シークレットを使う外部サービスは使わない
- 画面遷移・ルーティングは実装しない（単一画面）
