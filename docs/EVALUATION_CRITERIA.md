# EVALUATION CRITERIA

Evaluator が各 TASK を評価する際の基準です。
1つでも基準を満たさなければ **不合格** です。

---

## 1. 機能性

### 必須確認項目
- [ ] ユーザーが TASK の目的を達成できるか
- [ ] 主要操作（クリック・入力・遷移など）が実際に動作するか
- [ ] スタブ関数・ハードコード・仮データのまま完了扱いにしていないか
- [ ] 未接続の機能（ボタンがあるが何も起きない）がないか

### 不合格の例
- ボタンが存在するが `onClick` が未実装
- データが `null` や空配列のまま表示されている
- `console.log("TODO: implement")` が残っている
- 関数が `throw new Error("Not implemented")` のまま

---

## 2. UI品質

### 必須確認項目
- [ ] 見た目が破綻していないか（要素が重なる・はみ出すなど）
- [ ] 余白・色・文字サイズが自然か
- [ ] スマホ幅（375px）で表示が崩れないか
- [ ] テンプレート的な「AI っぽいデザイン」に偏りすぎていないか

### 不合格の例
- テキストがコンテナからはみ出している
- 375px 幅で横スクロールが発生する
- ボタンが小さすぎてタップできない（44px 未満）
- 文字が背景に埋もれて読めない

---

## 3. エラー耐性

### 必須確認項目
- [ ] 入力ミスや想定外の操作で UI が壊れないか
- [ ] ユーザーに分かるエラー表示があるか（エラーが起きた場合）
- [ ] ブラウザ API（Geolocation、カメラなど）が使えない場合でも落ちないか
- [ ] コンソールエラーが出ていないか

### 不合格の例
- Geolocation が拒否されたら白画面になる
- 空の入力でフォームを送信すると JS エラーがスローされる
- コンソールに `TypeError: Cannot read properties of undefined` が出ている

---

## 4. 実装品質

### 必須確認項目
- [ ] TypeScript の型が適切に定義されているか
- [ ] `any` 型を使っていないか
- [ ] `@ts-ignore` / `@ts-expect-error` を乱用していないか
- [ ] 不要な依存関係を追加していないか
- [ ] 大規模リファクタリング・TASK 範囲外の変更をしていないか
- [ ] `npm run build` が通っているか

### 不合格の例
- `const data: any = response` が含まれている
- TASK で変更すべきでないファイルが変更されている
- `package.json` に指示にない依存が追加されている

---

## 5. タスク分離

### 必須確認項目
- [ ] 1 TASK の完了条件をすべて満たしているか
- [ ] commit が 1 TASK 単位になっているか（複数 TASK をまとめて commit していないか）
- [ ] `docs/STATUS.md` と `docs/MVP_TASKS.md` が実態と一致しているか

### 不合格の例
- TASK-03 の commit に TASK-04 の変更が混入している
- STATUS.md が DONE になっているが完了条件を満たしていない
- MVP_TASKS.md のステータスが更新されていない

---

## Evaluator の判定フォーマット

```
## 評価結果: TASK-XX

**判定**: 合格 / 不合格

### 修正必須
- （不合格の場合のみ）

### 修正推奨
- （合格でも改善点があれば）

### 確認した操作
- 〇〇ページを開いた
- 〇〇ボタンをクリックした
- スマホ幅 375px で確認した

### Generator に戻す修正プロンプト（不合格時のみ）
---
TASK-XX の修正を行ってください。

問題:
- 〇〇が動作しない（再現手順: 〇〇）

修正してほしいこと:
- 〇〇を〇〇に修正する

完了条件:
- 〇〇が動作すること
---
```
